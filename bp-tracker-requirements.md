# 血圧記録アプリ 要件定義書 v1.0

> **本書の目的**: コーディングツール（Claude Code / Cursor 等）にそのまま読み込ませ、**一度の実装で完成**させるための完全仕様書。あいまいさを残さず、デザイン値・実装方針・落とし穴まで明記する。
> **実装者への指示**: 本書の「★最重要」「実装上の必須注意」は厳守すること。仕様外の独自解釈で機能を削らない。

---

## 0. 前提の確定（重要・先に読むこと）

依頼内には「携帯で入力したい」と「利用環境はまずPCブラウザ」の双方がある。これを次の通り確定する。

- **方針**: **レスポンシブWebアプリ**として実装する。**第一ターゲットはPCブラウザ（印刷の主担当）**、**入力はモバイルでも完全に快適**に行えること。
- **印刷はPCブラウザから実行**することを主導線とする（モバイル印刷も妨げない）。
- 「どんどんページを増やしていける」= **記録が増えるほど印刷が自動で複数ページに分割（ページネーション）される**こと、と解釈する。手動ページ追加機能は不要。
- バックエンド・サーバーなし。**完全ローカル動作・オフライン可・外部送信なし**。

---

## 1. 目的・背景

- **目的**: 血圧の記録、期間を指定した一覧表示、**医師提示用のA4縦・印刷レイアウト出力**。
- **背景**: 本人が日々測定値を記録し、通院時に印刷して医師に渡す。長期（数年）継続を前提とする。
- **成功の定義**: 「測ったら数秒で入力完了」「印刷ボタンで、はみ出し・余白過大のない読みやすいA4が出る」。

## 2. スコープ

### 2.1 対象（In Scope）
記録の追加 / 一覧表示 / 編集・削除 / 期間指定フィルタ / 日付順並び替え / 印刷用レイアウト表示 / ブラウザ印刷 / 設定 / **データのバックアップ（エクスポート・インポート）**。

### 2.2 対象外（Out of Scope）
クラウド同期 / 複数ユーザー / ログイン認証 / グラフ可視化 / 通知・リマインド / 医療的な診断や助言。（一部は §17 に将来拡張として記載）

## 3. 用語定義

| 用語 | 定義 |
|---|---|
| 収縮期血圧（SBP） | 最高血圧。単位 mmHg。整数。 |
| 拡張期血圧（DBP） | 最低血圧。単位 mmHg。整数。 |
| 脈拍（PR） | 1分あたりの拍数（bpm）。整数。 |
| レコード | 1回の測定＝1行（日付・SBP・DBP・PR・メモ・内部ID・登録時刻）。 |
| 印刷ビュー | 画面UI（ボタン等）を排除し、表のみをA4向けに整形した表示。 |

## 6. 機能要件

### 6.1 記録の追加
- 日付・SBP・DBP・PR を入力し、メモは任意。バリデーション通過時のみ保存。
- 保存後はトースト「保存しました」を表示し一覧へ戻る。
- **同一日に複数レコードを許可**（朝・夜など）。

### 6.2 一覧表示
- 全レコードを表で表示。列: 日付 / SBP / DBP / PR / メモ。
- 画面上は `120 / 80` のように SBP/DBP を併記表示してもよい（印刷表は列分割、§9）。
- 件数表示（例:「全 128 件 / 表示 24 件」）。空データ時はエンプティステート（§14）。

### 6.3 編集・削除
- 行タップで②に遷移し編集。削除は**確認ダイアログ必須**（「この記録を削除しますか？」）。

### 6.4 期間指定（フィルタ）
- 開始日・終了日の入力。**クイックプリセット**: 今週 / 今月 / 過去1ヶ月 / 過去3ヶ月 / 全期間。
- フィルタは一覧と印刷の双方に反映。未指定＝全期間。開始>終了は入力時に警告し抑止。

### 6.5 並び替え
- **日付順**。昇順／降順をトグル。画面の既定＝**降順（新しい順）**。
- 同一日に複数ある場合は**登録時刻（内部）で二次ソート**して順序を安定化。
- 印刷は既定で**昇順（古い順＝時系列）**を推奨（医師が経過を追いやすい）。トグルで変更可。

### 6.6 印刷用レイアウト表示・印刷 ★最重要
- 「印刷」押下で③印刷ビューを表示。`Ctrl/⌘+P` でブラウザ印刷。詳細仕様は **§9**。

### 6.7 設定
- 氏名（印刷ヘッダー表示用・任意）、印刷文字サイズ（縮尺）、ページ番号ON/OFF、ページ幅自動フィットON/OFF、並び順既定。

### 6.8 データのバックアップ（エクスポート / インポート）★必須追加
- **理由**: ローカル保存（localStorage）はブラウザのデータ削除で消える。数年分の医療記録を守るため必須機能とする。
- **エクスポート**: 全データを **JSON**（完全バックアップ）と **CSV**（Excel互換）でダウンロード。
- **インポート**: JSONを読み込み復元（既存とマージ or 置換を選択）。
- **全削除**: 二段階確認のうえ消去。

## 7. データモデル

### 7.1 レコード（必須項目）

| 項目 | キー | 型 | 必須 | 制約・既定 |
|---|---|---|---|---|
| 内部ID | `id` | string(UUID) | ○ | 自動生成 |
| 日付 | `date` | string `YYYY-MM-DD` | ○ | 既定＝今日 / 未来日不可（max=今日） |
| 収縮期血圧 | `sbp` | integer | ○ | 数字のみ / 50–300 |
| 拡張期血圧 | `dbp` | integer | ○ | 数字のみ / 30–200 |
| 脈拍 | `pr` | integer | ○ | 数字のみ / 30–250 |
| メモ | `memo` | string | × | 任意・空欄可 |
| 登録時刻 | `createdAt` | string(ISO) | ○ | 自動 / 二次ソート用 |

> 注: 脈拍を測定しない運用なら `pr` を任意化してよい（その場合は §10 の必須から外す）。既定は必須。

### 7.2 オプション項目（任意拡張・既定OFF・実装は将来でも可）
- `timing`（朝 / 夜）: 医師は朝晩の区別を重視するため、有効化推奨の余地あり。本書のコア要件には含めず、データ構造だけ拡張可能にしておく。

## 8. 入力UX仕様 ★（手間ゼロUI）

**設計原則**: 「測定値に数字を合わせて保存」だけで完了。スマホで親指だけで完結。

### 8.1 数値入力（SBP / DBP / PR 共通コンポーネント）
各フィールドは**3手段を併設**し、好きな方法で入力できる:
1. **大きな数値表示**（例 48–56px・等幅数字 `tabular-nums`）。
2. **タップでキーボード入力**（`inputmode="numeric"`、不正文字は入力時に弾く）。
3. **ホイールピッカー（ドラムロール式）**: 縦スクロールで数値選択・スナップ。
   - 範囲: SBP `70–250` / DBP `40–160` / PR `30–200`、刻み `1`。
4. **±ステッパー**（大きなタップ領域、刻み1、長押しで連続）。

- **スマートデフォルト**: 初期値＝**直近レコードの同項目値**（初回は 120 / 80 / 70）。ピッカーの初期位置も同値に合わせ、スクロール量を最小化。
- タップ領域は **最小 44×44px**。

### 8.2 日付
- ネイティブ日付ピッカー（`type="date"`）。既定＝今日、`max`＝今日。

### 8.3 メモ
- 1–2行のテキストエリア。任意。

### 8.4 保存
- 画面下部に**大きな固定「保存」ボタン**。押下→バリデーション→保存→トースト→一覧へ。
- バリデーションNG時はエラー箇所をハイライトし、保存を抑止（§10）。

## 9. 印刷レイアウト仕様 ★最重要

> **狙い**: A4縦で「余白過大なし・横はみ出しなし・行間詰めて可読・複数ページ自動分割・各ページに列見出し」。

### 9.1 ページ・余白（固定）
```css
@page { size: A4 portrait; margin: 12mm 12mm 14mm 12mm; }
```
- 印刷可能領域 ≈ 幅186mm × 高さ271mm。余白は固定（過大余白を作らない）。

### 9.2 表（ページ幅に収める／はみ出し防止）
```css
table { table-layout: fixed; width: 100%; border-collapse: collapse; }
thead { display: table-header-group; }      /* 列見出しを各ページ先頭に繰り返す */
tr    { break-inside: avoid; page-break-inside: avoid; } /* 行をページ間で分断しない */
th, td { border: 0.4pt solid #000; padding: 2px 4px; }  /* 行間を詰める */
td.num { text-align: center; font-variant-numeric: tabular-nums; }
td.memo{ overflow-wrap: anywhere; word-break: break-word; } /* 長文メモのはみ出し防止 */
```
- **列幅（合計100%／固定）**: 日付 24mm / SBP 20mm / DBP 20mm / PR 18mm / メモ 残り（可変・折り返し）。
- 罫線は **B&Wプリンタでも明瞭**な黒線。**色に依存しない**（画面の色分けは印刷では使わない、§13.6）。

### 9.3 ヘッダー・フッター
- **印刷ヘッダー（1ページ目上部）**: タイトル「血圧記録」 / 氏名（設定値・任意） / **期間「YYYY/MM/DD 〜 YYYY/MM/DD」** / 出力日。
- **列見出し**（日付・SBP・DBP・PR・メモ）は `thead` により**全ページに自動反復**。
- **フッターのページ番号（任意）**: 設定でON/OFF。実装方針は §9.5 を参照。

### 9.4 縮尺（自動調整）
- 設定で**文字サイズ**を選択: 標準 `10pt` / コンパクト `9pt` / 最小 `8pt`（`body` クラス切替で `font-size`・`padding` を変更）。
- **「ページ幅自動フィット」ON**: 横はみ出しが起きる場合に列幅・フォントを段階的に縮小（基本は `table-layout:fixed` + 折り返しで担保し、補助としてスケール調整）。

### 9.5 実装上の必須注意（手戻り防止）★
- **画面UI（ボタン・FAB・ナビ）は印刷で非表示**にする: `@media print { .no-print { display:none } }`。印刷ビュー本体のみ印刷。
- **ページ番号の落とし穴**: ネイティブ印刷（Chrome/Firefox/Safari）は CSS `@page` のマージンボックスでの `counter(page)` を**確実にはサポートしない**。ページ番号は以下のいずれかで実現する:
  - (A) **ブラウザ印刷ダイアログの「ヘッダーとフッター」をON**にする運用（追加実装ゼロ）。
  - (B) **Paged.js（MITライセンス）をローカル同梱**して `@page { @bottom-center { content: counter(page) " / " counter(pages) } }` を使用。完全カスタムの反復ヘッダー＋ページ番号＋正確なA4プレビューが可能。
- **推奨**: ベースは**ネイティブ `@media print`（依存ゼロ・完全オフライン）**で全要件を満たす。ページ番号を出力内に固定したい場合のみ (B) を採用。どちらでも本書の受け入れ条件は満たせる。

## 10. バリデーション仕様

| 項目 | ルール | NG時の挙動 |
|---|---|---|
| 日付 | 必須・`YYYY-MM-DD`・未来日不可 | 保存抑止＋「日付を入力してください」 |
| SBP | 必須・数字のみ・50–300 | 保存抑止＋該当欄ハイライト |
| DBP | 必須・数字のみ・30–200 | 保存抑止＋該当欄ハイライト |
| PR | 必須・数字のみ・30–250 | 保存抑止＋該当欄ハイライト |
| メモ | 任意 | 制限なし |
| 論理整合 | DBP ≥ SBP は通常あり得ない | **ソフト警告**（「最低≧最高です。よろしいですか？」）し、確認後は保存可 |
| 期間 | 開始 ≤ 終了 | 開始>終了は警告し抑止 |

- 共通: **空欄は保存不可（メモを除く）**。数値欄は**数字以外を入力段階で受け付けない**。

## 11. 保存・データ管理

- **保存先**: `localStorage`（単一ユーザー・小容量で十分。数千件でも上限内）。同期的で実装が簡潔。
- **スキーマ（バージョニング）**:
```json
{
  "version": 1,
  "settings": { "name": "", "printFontSize": "standard", "showPageNumber": false, "autoFit": true, "defaultSort": "desc" },
  "records": [
    { "id": "uuid", "date": "2026-06-20", "sbp": 124, "dbp": 78, "pr": 66, "memo": "起床後", "createdAt": "2026-06-20T07:10:00+09:00" }
  ]
}
```
- **書き込み**: 追加・編集・削除のたびに保存。`try/catch` で失敗時はトースト警告。
- **復旧**: 起動時のJSONパース失敗に備え `try/catch`。破損時は空で起動しつつ「バックアップから復元してください」を案内（§6.8）。
- **プライベートブラウジング等でlocalStorage不可**の場合は、保存不可を明示し、エクスポート利用を促す。

## 12. 非機能要件

- **ローカル動作・オフライン**: 起動後はネットワーク不要。**実行時に外部リソースへ通信しない**（ライブラリ・フォントは同梱）。
- **プライバシー**: **データの外部送信は一切しない**。解析・トラッキングなし。
- **性能**: 数千件のレコードでも一覧・フィルタ・印刷が体感ストレスなく動作。
- **アクセシビリティ**: 主要操作はキーボード可。コントラスト比 AA 以上。タップ領域 ≥44px。文字は十分な可読サイズ。
- **対応ブラウザ**: 最新版 Chrome / Edge / Safari / Firefox。

## 13. デザイン仕様 ★（UI/UX）

**トーン**: 清潔・医療的な信頼感・落ち着き。情報密度は高すぎず、入力は大きく明快に。

### 13.1 カラーパレット
| 用途 | 値 |
|---|---|
| 背景 | `#F4F6F8` |
| サーフェス（カード） | `#FFFFFF` |
| プライマリ（信頼の青） | `#2563EB` / hover `#1D4ED8` |
| テキスト主 / 副 | `#111827` / `#6B7280` |
| 罫線（画面） | `#E5E7EB` |
| エラー・高値 | `#DC2626` |
| 成功・正常 | `#16A34A` |
| 注意 | `#D97706` |

### 13.2 タイポグラフィ
- `font-family: system-ui, "Hiragino Sans", "Noto Sans JP", "Meiryo", sans-serif;`
- 本文 15–16px。**数値表示は大きく太字＋ `tabular-nums`** で桁を揃える。

### 13.3 レイアウト・間隔
- **8pxグリッド**（4 / 8 / 12 / 16 / 24 / 32）。角丸: カード12px・ボタン/入力8px。影は控えめ `0 1px 3px rgba(0,0,0,.08)`。
- 一覧の右下に **FAB「＋」**（記録追加）。主要操作は親指の届く下部に配置。

### 13.4 コンポーネント定義
- **ボタン**: プライマリ（塗り）/ セカンダリ（枠線）/ 破壊的（赤・削除）。
- **入力フィールド**: §8の数値コンポーネント、日付ピッカー、テキストエリア。
- **テーブル**: 画面用（読みやすい行高）/ 印刷用（行間詰め、§9）。
- **トースト**: 保存完了・エラー。**確認ダイアログ**: 削除・全削除。

### 13.5 状態
- **エンプティ**: 「まだ記録がありません。右下の＋から最初の記録を追加しましょう。」
- **ローディング/エラー/トースト** を各々定義。

### 13.6 色分け（任意・画面のみ・医療助言ではない）
- 一覧で値帯を**淡色背景**で示してよい（あくまで視認補助）。例（家庭血圧の目安・JSH2019）: 〜134/84 緑系 / 135–159・85–99 黄系 / 160以上・100以上 赤系。
- **注記必須**: 「色分けは目安であり診断ではありません」。**印刷では色に依存しない**（B&Wで読める設計、§9.2）。

## 14. エラー処理・エッジケース

- **同一日複数レコード**: 許可。`createdAt` で安定ソート。
- **長文メモ**: 画面は折り返し、印刷は `overflow-wrap:anywhere` で枠外にはみ出させない。
- **期間外/不正範囲**: 開始>終了は抑止。該当0件は「条件に合う記録がありません」。
- **未来日**: 入力時に抑止（max=今日）。
- **localStorage不可/満杯/破損**: 警告表示＋エクスポート/インポート誘導（§11）。
- **印刷のページまたぎ**: 行を分断しない・列見出しを各ページ反復（§9.2）。

## 15. 受け入れ条件（テスト可能チェックリスト）

- [ ] レコードを追加でき、一覧に表示される。
- [ ] 期間指定（直接入力・プリセット）で正しく絞り込める。
- [ ] 日付順（昇順/降順）で並ぶ。同一日複数も順序が安定。
- [ ] 数値欄は数字のみ受け付け、**空欄は保存不可**、**日付は必須**。
- [ ] レコードの編集・削除（確認ダイアログ）ができる。
- [ ] **リロード後もデータが保持される**（localStorage）。
- [ ] エクスポート（JSON/CSV）・インポートでデータを復元できる。
- [ ] **印刷プレビューで余白過大・横はみ出しがない**。
- [ ] **A4縦で読みやすく出力**でき、表がページ幅に収まる。
- [ ] 複数ページに自動分割され、**各ページに列見出しが出る**。**行がページ間で分断されない**。
- [ ] 印刷ヘッダーにタイトル・期間（・氏名）が出る。ページ番号はON/OFFできる（運用 or Paged.js）。
- [ ] 印刷文字サイズ（縮尺）と自動フィットが設定で切り替わる。
- [ ] スマホで、ピッカー・ステッパー・キーボードのいずれでも数値入力でき、数秒で保存できる。

## 16. 推奨技術スタックと実装ガイド

- **推奨構成**: **依存最小の単一ページWebアプリ**。
  - 第一推奨: **素のHTML + CSS + JavaScript（単一 `index.html`、ビルド不要）**。印刷CSSの制御性が最も高く、完全オフライン・高い可搬性。ダブルクリックで起動。
  - 代替: **React + Vite**（コンポーネント分割を好む場合）。ただし印刷の `@media print` 制御を最優先で検証すること。
- **ライブラリ**:
  - 基本は**ライブラリなし**で実現可能。
  - ページ番号を出力内に固定したい場合のみ **Paged.js を同梱**（CDN実行時依存は避け、ファイルをローカルに置く＝オフライン/プライバシー要件のため）。
- **印刷実装の優先順位**: ①ネイティブ `@media print`（§9）でまず完成 → ②ページ番号要件が厳しければ Paged.js を追加。
- **データ層**: `localStorage` を §11 のスキーマで実装。エクスポート/インポートを最初から組み込む。
- **やってはいけないこと**: 実行時の外部API通信 / データの外部送信 / 印刷で色に依存する設計 / 入力欄を1手段に限定すること。
